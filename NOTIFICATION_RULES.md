# 通知判定ルール仕様書

> **この文書が唯一の人間向けルール一覧。** ルールの実体は config.py / check_stock.py に実装され、
> tests/test_logic.py がCIで毎回検証する。ルールを変更したら必ずこの文書も更新すること。
> 基本思想: **「通知が来たら本物」を守る。迷ったら通知しない側に倒す。**

## 1. 何を通知するか（共通・3通知経路すべてに適用）

通知経路 = ①ページ差分通知（リアルタイム） ②毎朝のチャンスダイジェスト ③RSS発見

| # | ルール | 実装 | 由来 |
|---|-------|------|------|
| 1 | 行動語（再販/入荷/抽選/予約/受付/先着/販売/在庫/整理券/受注/コラボ/限定/プロモ/発売）を含む行だけ通知 | `NOTIFY_ACTION_KEYWORDS` | PR #10,16 |
| 2 | 行動語がなくても「45日以内の将来日付＋日付以外の中身8字以上」なら通知（受付期間の更新） | `_is_actionable_line` | PR #11,15 |
| 3 | 日付だけの行（例:「2026.7.10」）は通知しない — 情報ゼロ | 同上 | PR #15 |
| 4 | 行の長さ 8〜120字（短い断片・長文記事は除外） | `DIGEST_LINE_MINLEN`/120 | PR #14,15 |
| 5 | 定型文・ナビ断片（収録カードリスト/当たりカード/買取価格/封入確率/ストア一覧/随時更新中/速報をお届け/コチラ等）は除外 | `DIGEST_EXCLUDE_MARKERS` | PR #10,15 |
| 6 | 状態継続文言（販売継続中等）は「変化」でないため通知しない | `STATUS_QUO_MARKERS` | PR #23 |
| 7 | ページ更新でも実質情報の行が増えていなければ通知抑制（削除だけ・定型文変化は無通知） | `_process_item` | PR #10 |

## 2. 商品ライフサイクル規則（ユーザーのドメイン知識・2026-07-10〜14）

| # | ルール | 実装 | 由来 |
|---|-------|------|------|
| 8 | **対象はカードゲーム商品のみ**。RSSキーワードもカード語限定（素の「ワンピース」等は使わない） | `WATCH_KEYWORDS` | PR #17 |
| 9 | サプライ類（デッキケース/スリーブ/プレイマット等）は常に通知しない | `SUPPLY_NOISE_KEYWORDS` | PR #15,16 |
| 10 | スターターセット/構築デッキ系は**ポケカのみ・初回販売（発売前後60日）だけ**通知。再販は通知しない。ポケカ以外は通知しない | `DECK_PRODUCT_KEYWORDS` `INITIAL_SALE_WINDOW_DAYS` | PR #16 |
| 11 | **例外: スタートデッキは再販でも人気** → 初回も再販も通知 | 同上（除外リスト外） | PR #16 |
| 12 | **発売から1年半（548日）超の商品は追わない**（全商品共通・再販が来ないため）。監視の自動失効＋通知行の除外＋監視候補提案の除外＋相場評価停止＋東映スイープにも適用 | `MAX_PRODUCT_AGE_DAYS` `release_date` | PR #16,17,18 |
| 13 | 非カード商品タグ（【雑誌】【書籍】【フィギュア】【ぬいぐるみ】【CD】【DVD】【ゲーム】等）は原則通知しない | `NON_CARD_CATEGORY_TAGS` | PR #23 |
| 14 | **例外: カード付録つき雑誌**（Vジャンプのプロモカード等）は通知する（プロモカード/カード付/付録カード/特典カード等で判定） | `MAGAZINE_CARD_MARKERS` | PR #23 |

## 3. リンクの付け方（「通知だけで完結」）

| # | ルール | 実装 | 由来 |
|---|-------|------|------|
| 15 | 直リンクは**確実な対応が取れた場合だけ**: ①行がアンカーテキストそのもの（15字以上の完全包含） ②行内の店舗名とURLドメインが一致 | `resolve_store_link` | PR #13 |
| 16 | 確実な直リンクが無い行は「→検索:」でストア検索URL（商品名クエリ・店舗名があればそのストア） | `fallback_search_url` | PR #14 |
| 17 | アフィリエイトURLは剥がして実URL化（&amp;エンティティ復元込み）。ストアドメイン19種以外のURLは載せない | `_unwrap_affiliate` `STORE_DOMAINS` | PR #10,11,13 |
| 18 | RSS発見は記事ページから店舗タグ一致リンクを解決して併記（記事URLも保持） | `resolve_store_link_from_article` | PR #19 |

## 実装モジュール構成（2026-07-14リファクタリング後）
- `check_stock.py` — 監視の実行・状態管理・通知（オーケストレーション）
- `rules.py` — 判定規則（日付解決・実質情報・商品ライフサイクル・相場選別）
- `links.py` — リンク解決（アンカー照合・アフィ剥がし・検索URL）
- `netutil.py` — 通信（サーキットブレーカー付きHTTP）
- `config.py` — 監視対象と全ルールの定数
